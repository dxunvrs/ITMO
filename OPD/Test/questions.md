### Операциионные системы
Замена оператора работы компьютерной системы  
Делятся на пользовательские, серверные и встроенные ОС

### История UNIX/Linux
Сначалы был Unix  
От него появились BSD и PWB/UNIX  
Через время от PWB/UNIX произошли закрытые ОС, например, Solaris, AIX, SunOS  
На базе BSD было создано FreeBSD, NetBSD, OpenBSD  
Также на базе FreeBSD появилась Mac OS X  
Независимо от Unix появился Linux и Minix  

### Современность

BSD - Mac OS X, NetBSD, FreeBSD, OpenBSD  
Linux - Ubuntu, SUSE, RedHat, Fedora, ...

### Архитектура ЭВМ
Гарвардская архитектура - инструкции и данные отдельно  
Instrucion memory <-> Control Unit <-> Data Memory  
ALU <-> Control Unit <-> I/O  
Неймана - инструкции и данные вместе  
Input -> CPU(состоит из Control Unit и ALU) -> Output  
CPU <-> Memory Unit

### АЛУ, коммутатор, блок признаков резльутата

На выходе блок проверки и установки признаков результата, который ставит 4 флага в регистр PS - N,Z,V,C

### Представление чисел: фиксированная точка
Фиксируем точку за каким-то разрядом, все что до нее имеет отрицательный вес  

### Представление чисел: беззнаковые целые
Всего возможно 0<=X<=2^n-1, где n - количество разрядов  

### Представление чисел: знаковые целые
Прямой код - оставляем представление такое же,но добавляем знаковый бит, минусы - есть двойной ноль  
Доп. код - M=b^n-K, где M - наше дополнение, b - основание СС, n - кол-во разрядов  
Обратный код - инверсия представления, минусы - двойной ноль  

### Представление чисел: доп.код
M=b^n-K=((b^n-1)-K)+1
Прелесть в том, что (b^n-1)-K это дополнение циферок числа до максимальной цифры(например в 16СС, A->5, 4->C)  
Остается добавить +1, именно так БЭВМ и делает  

### Перенос, переполнение
Сложили два положительных, а получиили отрицательное - переполнение  
Перенос - выход за разрядную сетку  
Важный момент: в беззнаковой арифметике перенос - это переполнение  

### БЭВМ: представление чисел
БЭВМ использует доп.код  
ОДЗ: 0<=X<=2^16-1 для беззнаковых и -2^15<=X<=2^15-1 для знаковых  

### Представление чисел с плавающей точкой
БЭВМ - нет  
А так мировой стандарт - IEEE 754  
X = (-1)^sign * (1+Mantissa) * 2^exp-127

### Символы: UNICODE, UTF-8
Unicode - единый способ кодирования символов всех языков мира, у каждого символа свой уникальный идентификатор  
UTF-8 - форма представления Unicode, использует переменное количество байт (от 1 до 4)  
Если 1 байт, то первый будет 0xxxxxxx  
Если 2 байта, то 110xxxxx  
Если 3 байта, то 1110xxxx  
Если 4 байта, то 11110xxx  

### Big-endian и Little-endian 
Little-endian - порядок от младшего к старшему (например 123 как 321), используется на x86 архитектуре  
Big-endian - порядок от старшего к младшему, стандартный порядок для многих протоколов(например, TCP/IP)  

### Представление строк

2. Упаковка с длиной - сначала длина, потом сами символы

### Канальная организация
По шинам все идет с устройства через память в CPU, из CPU все так же через память на вывод  

### Раздельные шины
C устройства на специальные шины, с оттуда на CPU, CPU общается с памятью через отдельные шины, дальше с CPU на все теже шины ввода-вывода на выход  
Шины адреса для ввода-вывода, шины данных для ввода-вывод и шины управления для ввода-вывода раздельны  

### Общие шины
Все шины объединены, работа с памятью через них же  

### Мультиплексирование шин
Шина адреса и данных с шиной управления раздельны, работа с памятью через те же шины, что и для ввода-вывода  

### Мультипроцессорность: UMA - Uniform Memory Access
Есть системная шина, на нее идут несколько CPU, банков памяти и контроллеров управления  

### Мультипроцессорность: Коммутатор
Вместо системной шины - Crossbar коммутатор  
Также добавлен IO мост для разделения работы с вводом и выводом  

### Мультипроцессорность NUMA - Non Uniform Memory Access
Несколько отдельных системных плат, есть плата с CPU-шками и банками памяти, отдельно платы с памятью отдельно плата с вводом-выводом, связь все также через Crossbar коммутатор

### CISC, RISC, VLIW
- Complex Instruction Set Computer - например Intel  
- Reduced Instrucion Set Computer - простой набор инструкций, выполнение инструкции за такт  
- Very Long Instructions Word - несколько инструкций, упакованных в одну команду, упаковка операций в инструкцию ложится в компилятор
